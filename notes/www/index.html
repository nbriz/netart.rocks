the World Wide Web

What was often difficult for people to understand about the design was that there was nothing else beyond URIs (URLs), HTTP, and HTML. There was no central computer controlling the Web no single network on which these protocols worked, not even an organization anywhere that ran the Web. The Web was not a physical thing that existed in a certain place, it was a space in which information could exist.

Tim Berners-Lee (from Weaving the Web)

The Web is an open and globally decentralized document and application distribution platform. Documents (ie. web pages) and software (ie. web apps), written in HTML, CSS, JavaScript and other languages, are stored on computers (connected to the Internet) all around the world running applications called servers, which wait for HTTP requests (small packets of data asking for specific files or resources) coming from other applications known as "clients" or "agents" (which are typically web browsers or web crawlers).

Anyone can create a website or web app and host their files on their own server running on their own computer connected to the Internet, and anyone else can use any number of web clients to request (ie. access) that website or web app over the Internet. As a platform, the Web has no central server, it's a collection of servers running on computers all over the world, which is what we mean when we say that the web is a "decentralized" platform. You don't need to ask anyone for permission or approval to run a server on the Web, all you need is a computer with an Internet connection, this is why we say the Web is an "open" platform.

While no one owns and operates the Web (the platform) itself, the protocols (like HTTP), core languages (HTML and CSS) and JavaScript APIs are designed and maintained by an internatoinal organization called the W3C, or the World Wide Web Consortium which consists of businesses, nonprofit organizations, universities, governmental entities, and individuals (anyone can join). The W3C don't own or control the Web (again, it's an open and decentralized platform), instead they're in charge of describing how it should work. These descriptions take the form of documents known as "specifications" (or "specs" for short) which are used as blue-prints for anyone interested in writing their own web server or web client software, as well as anyone interested in creating their own websites or web apps. If you don't follow the spec, then your client, server, site or app won't be compatible with the platform (ie. it won't work).




Web Clients (aka "browsers")

the apps we use to access the Web

As mentioned earlier, a web client is an application that can send HTTP requests to web servers and render the code it gets back. Some web clients are designed only to send requests to specific servers (like most of the social media apps you have on your phone for example), while others can send requests to any server you specify (like any web browser for example), usually in the form of a URL or "web address".


Firefox is a free and open-source web browser developed by the Mozilla Foundation, a non-profit working to ensure the internet remains a public resource that is open and accessible to us all.




Code Editor

creating the Web

A website and web app consists of a series of files, some of these are "assets" (like individual images, videos, fonts, etc) while other files are written in code. We can use all sorts of different coding languages to create a website or web app, but the most fundamental is HTML (Hypertext Markup Language). HTML is the language we use to structure our site's content (whether that's text and/or other media assets). When we want to change the way our HTML files look, we use another language called CSS (Cascading Style Sheets). CSS code allows us to change a websites's style describe the layout and aesthetic we want. HTML and CSS can do a lot on their own, including some basic interactivity and animation, but when we really want to bring a website or app to life with even more interactive or generative dynamics we use JavaScript, the Web's de facto programming language.

Like all human languages, these coding languages have their own vocabulary and syntax. When you become literate in these languages you can use them to express and create anything you've ever seen on a computer screen. Like any other human language, coding languages are just text and so you can use any application capable of typing text to write code. That said, special programs called code editors make it a little easier to do so by adding special features like code hinting (aka auto-complete), syntax highlighting (adding colors to different parts of our code to make it easer to read/write) and shortcut keys. A few popular contemporary code editors include Sublime, Atom and VS Code.

Web Servers

publishing to the Web

A web "host" is a service running their own web servers on their own computers that you can use to host the websites or apps you create. While it's totally possible to host your site/app at home by running your own web server on your own computer, it isn't always practical (unless you've got lots of bandwidth, great upload Internet speeds and a computer you don't mind leaving on/connected running your server 24/7). For this reason there exist all sorts of different services for hosting your websites/app s on someone else's computer (aka on "the cloud"). Most of these servers will rent you space on their computers/data centers for a monthly fee, but there are also some "free" hosting providers out there.


a few of the the Internet Archive's servers; from a short documentary about the Archive


One place we can publish (or "host") our documents for free is GitHub, a platform created to facilitate "open source" development, where coders can share their code and collaborate with each other. Developers share their source code openly this way for a variety of reasons, some are practical some are ideological, but some developers and corporations prefer to keep their code secret. When the code used to create a site or application is kept secret it's known as "proprietary" software. In the interest of transparency, accessibility and to open the door for future participation, the documents we'll produce will be open source. Though GitHub isn't the only platform for sharing open source code, it's unquestionably the most popular (at the time of this writing) and additionally provides the ability to run a free Web server which we can use to "host" our documents online.

To learn more about the history of "open source" culture, refer to my FLOSS notes (Free/Libre Open Source Software)
