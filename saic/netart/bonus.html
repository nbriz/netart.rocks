RE:META

There is not a single person in existence who has scanned Facebook’s News Feed and said: Yes, immerse me in this reality. I want to feel my uncle’s meme about Hot Pockets on my face. But “the metaverse” could generate enough momentum, enough knock-on interest, that it could bring this clumsy fantasy framework clattering to life. Which is exactly why this half-real, Big Tech–led effort to erect the metaverse is worth both laughing down and taking seriously.

Brian Merchant
The Real Reason Facebook Changed Its Name (The Atlantic)

I'm obviously biased, but it hurts to see Facebook rebrand itself using a term with an otherwise inspiriting history in this context. Under Meta's reign, the future of mixed reality is likely to look a lot less like the optimistic visions presented by the artists and technologists which paved the way, and much more like the dystopian futures articulated by cyberpunk authors.

Still, it's not a foregone conclusion that a single surveillance capitalist corporation will colonize cyberspace as it has other virtual spaces. As I write this there are loads of artists exploring the emerging technology of virtual reality as a tool as well as an artistic medium, artists like Li Alin or Jordan Wolfson's controversial piece "Real Violence" exhibited at the Whitney Biennial, or this VR Dance piece Dust by Mária Júdová and Andrej Boleslavský and this VR immersive dining experience The Aerobanqueets RMX by Mattia Casalegno, just to name a few (for more check out these portals, publications and online and offline VR galleries). Like all new technologies, there exists lots of potential promise (consider how VR is being used in society today: from education, to surgical training, as evidence in court cases and for physical therapy and mental health) as well as potential peril.

As artists we need to remain critical of the way a toxic corporations like Facebook encroach on this space. We have role to play in both warning society of what this future "metaverse" could become, while also reminding society of all the more interesting and enriching alternative directions this can take. But in order to do that, it would help to learn how to engage with this new medium directly.

A-Frame

When Facebook talks about creating the "metaverse", it's talking about throwing fencing around a technology that already exists on the Web as it's done it's entire history, in this instnace we're talking about WebXR (formally WebVR) which is how you create mixed (virtual and augmented) reality "experiences" on the Web. The easiest way to get started is probably using the A-Frame library, which allows you to create 3D/VR environments using custom HTML elements. The library itself is well documented but I also have notes available from a workshop I've given at various schools in the past.

Additionally, there are a few netnet tutorials on cyberspace, the first tutorial covers some of the fundamental concepts associated with the medium as well as the history of it's development and the various ways artists have engaged with it over the years, while subsequent tutorials cover the craft, specifically how to create a scene, working with assets and ends with a review, but BE WARNED, this is an older version of netnet (BETA-1.0), it doesn't have the same features/feel as the version you're used to (and is likely very buggy).

Three.js

A-Frame is build on top of another very popular library called Three.js, a JavaScript library for creating 3D work on the Web. This is something I cover a little bit in the Internet Art 2 course, but ultimately requires you know how to code in JavaScript as well as some familiarity with Object Oriented Programming. A while back I created an online platform for experimenting with the library called the three.js_playGnd without having to learn too much JavaScript first. Sadly that platform is now a bit outdated, but there are loads of other (newer) tutorials out there, my favorite these days is this online course.
